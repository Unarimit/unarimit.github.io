# 博客写作风格指南

本文档总结了现有技术博客的写作风格，用于在后续自动化或协作任务中保持风格统一。

## 结构规范

1.  **主标题**：每篇文章以一个一级标题 (`#`) 开始，作为文章的主题。
2.  **章节划分**：使用二级 (`##`) 和三级 (`###`) 标题来组织文章内容，形成清晰的层次结构。
3.  **参考章节**：在文章末尾必须包含一个名为 `## 参考` 的二级标题，用于列出所有引用的外部链接、文章或文档。
4.  **列表**：使用无序列表 (`-`) 或有序列表 (`1.`) 来阐述观点、步骤或特性，使内容更具条理性。
5.  **图文结合**：在适当的位置插入图片 (`&lt;img src='...'&gt;`) 来辅助说明复杂的概念。

## 内容规范

1.  **技术深度**：文章内容应具有一定的技术深度，旨在解释清楚特定的开发概念或技术实现。
2.  **代码示例**：在解释技术点时，应提供简洁明了的代码示例。代码块需要使用 ``` 符号包裹，并明确标注语言类型（如 `cs`, `cpp`, `json` 等）。
3.  **内部链接**：鼓励在文章中通过相对路径链接到站内的其他相关文章，构建知识网络。例如：`[链接文本](../Category/Document.md)`。
4.  **外部链接**：对于关键的API、技术术语或概念，应提供指向官方文档或权威解释的超链接，方便读者深入学习。

## 语言风格

1.  **专业简洁**：使用专业、准确、简洁的语言进行描述。
2.  **中英混用**：在涉及技术术语、API名称或专有名词时，可以直接使用英文，无需翻译，以保证准确性。
